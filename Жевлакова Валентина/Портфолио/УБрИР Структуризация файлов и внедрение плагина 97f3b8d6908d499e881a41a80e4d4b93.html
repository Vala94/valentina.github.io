УБрИР. Структуризация файлов и внедрение плагина

Обложка

Задача

  1. Актуализировать макеты и создать понятную структуру хранения файлов
  1. Сократить нагрузку на дизайнеров по поиску актуальных макетов

Проблема

  1. Когда я пришла в банк не было никакой структуры хранения. Огромное количество макетов располагалось в двух файлах. Перед каждой задачей дизайнер вместе с лидом тратили большое количество времени на поиск актуальных макетов
  1. Аналитики, ПО и другие члены команды постоянно отвлекали дизайнеров от работы на поиск актуальных макетов

Решение


Структура хранения и актуализация:

  1. Распределила все актуальные макеты на файлы по разделам приложения

  1. Добавила дополнительные файлы для хранения важной информации

  1. Добавила файл с руководствами для дизайнеров

  1. Создала архивы для хранения артефактов

  1. Файлы для работы дизайнеров в зависимости от платформы

  1. Библиотеки компонентов

  1. В файлах с разделами приложения организовала странички для разных платформ и архивы

  1. Провела работу по актуализации макетов совместно с аналитиками, задала сетку и правила размещения сценариев с макетами. Теперь у каждого сценария стоит актуальный статус, номер задачи, дата создания макетов и дизайнер.

Внедрение плагина:

  1. Внедрила плагин, который позволяет автоматически создавать каталог и производить нумерацию макетов

  1. Для заинтересованных сотрудников каталог размещается на специальном сайте для самостоятельного поиска необходимых сценариев

  1. В плагине и на сайте с каталогом можно производить поиск по названию сценария, номеру задачи, по дате и по дизайнеру. Так же в каталоге отображается статус сценария

Обновление каталога на сайте производится автоматически при запуске дизайнером плагина
